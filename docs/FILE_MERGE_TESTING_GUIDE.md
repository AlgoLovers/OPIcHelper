# 🎯 파일 병합 테스트 가이드

## 📋 개요
암기레벨 영작테스트에서 녹음 파일들이 올바르게 병합되는지 확인하기 위한 테스트 방법

## 🧪 테스트 종류

### 1. 단위 테스트 (Unit Test)
- **파일**: `AudioFileMergeUnitTest.kt`
- **목적**: AudioFileManager의 병합 기능을 격리된 환경에서 테스트
- **장점**: 빠른 실행, 정확한 검증, 자동화 가능

### 2. 통합 테스트 (Integration Test)
- **파일**: `AudioFileMergeTest.kt`
- **목적**: 실제 앱 환경에서 전체 영작테스트 과정을 통한 병합 테스트
- **장점**: 실제 사용 시나리오 검증, UI 상호작용 포함

### 3. 기존 테스트
- **파일**: `EnglishWritingTestMergedFileTest.kt`
- **목적**: 병합된 파일의 재생 기능 테스트
- **장점**: 재생 기능까지 포함한 전체 플로우 검증

## 🚀 테스트 실행 방법

### 1. 전체 테스트 실행
```bash
# 모든 병합 관련 테스트 실행
./scripts/run-merge-tests.sh

# 특정 디바이스에서 실행
./scripts/run-merge-tests.sh [device_id]
```

### 2. 개별 테스트 실행
```bash
# 단위 테스트만 실행
./gradlew testDebugUnitTest --tests "*.AudioFileMergeUnitTest"

# 통합 테스트만 실행
./gradlew connectedAndroidTest -Pandroid.testInstrumentationRunnerArguments.class=com.na982.opichelper.presentation.ui.AudioFileMergeTest

# 기존 테스트 실행
./gradlew connectedAndroidTest -Pandroid.testInstrumentationRunnerArguments.class=com.na982.opichelper.presentation.ui.EnglishWritingTestMergedFileTest
```

### 3. Android Studio에서 실행
1. 테스트 파일 우클릭
2. "Run 'TestName'" 선택
3. 결과 확인

## 📊 테스트 검증 항목

### ✅ 파일 생성 확인
- [ ] 개별 녹음 파일이 생성되는가?
- [ ] 파일 크기가 0이 아닌가?
- [ ] 파일명이 올바른 형식인가?

### ✅ 파일 병합 확인
- [ ] 병합 과정에서 에러가 발생하지 않는가?
- [ ] 최종 병합 파일이 생성되는가?
- [ ] 병합된 파일 크기가 적절한가?

### ✅ 파일 형식 확인
- [ ] 병합된 파일이 올바른 M4A 형식인가?
- [ ] 파일 헤더가 정상적인가?
- [ ] 파일이 재생 가능한가?

### ✅ 성능 확인
- [ ] 병합 시간이 적절한가? (60초 이내)
- [ ] 메모리 사용량이 적절한가?
- [ ] CPU 사용량이 과도하지 않은가?

## 🔍 테스트 결과 분석

### 1. 성공 케이스
```
✅ 파일 병합 성공
- 개별 파일: 3개 (2048, 1536, 1024 bytes)
- 병합 파일: 4567 bytes
- 병합 시간: 2.3초
- 파일 형식: M4A (유효)
```

### 2. 실패 케이스
```
❌ 파일 병합 실패
- 원인: MediaCodec 지원 안됨
- 대안: Fallback 방식 사용
- 최종 결과: 성공
```

### 3. 성능 이슈
```
⚠️ 병합 시간 초과
- 예상 시간: 10초
- 실제 시간: 65초
- 원인: 대용량 파일 처리
```

## 🛠️ 문제 해결

### 1. 테스트 실패 시 확인사항
- **디바이스 연결**: ADB 연결 상태 확인
- **권한 문제**: 마이크 권한 확인
- **저장소 공간**: 충분한 공간 확보
- **네트워크**: 인터넷 연결 확인

### 2. 병합 실패 시 확인사항
- **개별 파일 상태**: 각 파일이 올바르게 생성되었는지
- **파일 형식**: M4A 형식이 올바른지
- **MediaCodec 지원**: 디바이스에서 MediaCodec 지원 여부
- **메모리 부족**: 충분한 메모리 확보

### 3. 성능 이슈 시 확인사항
- **파일 크기**: 개별 파일 크기가 과도하지 않은지
- **파일 개수**: 병합할 파일 개수가 적절한지
- **디바이스 성능**: 테스트 디바이스 성능 확인
