# 기본 동작 시나리오 (Basic Operations)

이 문서는 OPicHelper 앱에서 반드시 보장해야 하는 핵심 기능들을 정의합니다.
각 시나리오는 자동화된 테스트로 검증되어야 합니다.

## 1. 기본 앱 동작 (Core App Operations)

### 1-1. 앱 생명주기 관리 (App Lifecycle Management)

#### 1-1-1. 앱 실행 시 초기화
- **시나리오**: 앱이 처음 실행될 때
- **기대 동작**: 
  - 첫 번째 카테고리의 첫 번째 QA 아이템이 화면에 표시됨
  - 질문과 답변이 정상적으로 로드됨
  - 로딩 상태가 적절히 처리됨
  - 이전 세션의 상태가 복원됨 (선택된 카테고리, 암기 레벨 등)

#### 1-1-2. 앱 종료 시 상태 저장
- **시나리오**: 앱이 종료될 때 (백키, 홈키, 최근 앱 목록에서 제거)
- **기대 동작**:
  - 현재 진행 중인 암기 테스트 상태가 저장됨
  - 현재 카테고리와 QA 아이템 인덱스가 저장됨
  - 선택된 암기 레벨이 저장됨
  - TTS 재생이 정지됨

#### 1-1-3. 백그라운드 전환
- **시나리오**: 앱이 백그라운드로 이동할 때
- **기대 동작**:
  - 현재 진행 중인 TTS 재생이 정지됨
  - 암기 테스트가 중단됨
  - 현재 상태가 임시 저장됨

#### 1-1-4. 포그라운드 복귀
- **시나리오**: 앱이 백그라운드에서 포그라운드로 복귀할 때
- **기대 동작**:
  - 저장된 상태가 복원됨
  - UI가 이전 상태로 복원됨
  - 자동 재생은 시작되지 않음

### 1-2. 카테고리 관리 (Category Management)

#### 1-2-1. 카테고리 변경
- **시나리오**: 사용자가 다른 카테고리를 선택할 때
- **기대 동작**:
  - 선택된 카테고리의 QA 리스트가 정상적으로 로드됨
  - 해당 카테고리의 첫 번째 QA 아이템이 화면에 표시됨
  - 이전 카테고리의 상태가 정리됨
  - 진행 중인 암기 테스트가 중단됨

#### 1-2-2. 카테고리별 QA 아이템 이동
- **시나리오**: 이전/다음 스크립트 버튼을 클릭할 때
- **기대 동작**:
  - 이전 버튼 클릭 시 이전 QA 아이템으로 이동
  - 다음 버튼 클릭 시 다음 QA 아이템으로 이동
  - 첫 번째/마지막 아이템에서 경계 처리
  - 진행 중인 재생이 정지됨

### 1-3. 암기 레벨 관리 (Memorization Level Management)

#### 1-3-1. 암기 레벨 선택
- **시나리오**: 사용자가 암기 레벨을 선택할 때
- **기대 동작**:
  - 선택된 레벨이 UI에 반영됨
  - 버튼 텍스트가 선택된 레벨에 따라 동적으로 변경됨
  - 이전 진행 상태가 초기화됨

#### 1-3-2. 암기 레벨별 버튼 텍스트 변경
- **시나리오**: 반복듣기/영작테스트/통암기 레벨 선택 시
- **기대 동작**:
  - "반복 듣기" 선택 시 버튼 텍스트가 "반복듣기"로 변경
  - "영작 테스트" 선택 시 버튼 텍스트가 "영작테스트"로 변경
  - "통암기" 선택 시 버튼 텍스트가 "통암기"로 변경

### 1-4. TTS 재생 관리 (TTS Playback Management)

#### 1-4-1. 질문 재생
- **시나리오**: 질문 재생 버튼을 클릭할 때
- **기대 동작**:
  - 질문 TTS가 재생됨
  - 재생 중 정지 버튼 클릭 시 TTS가 정지됨
  - 하이라이트 기능이 정상 동작함
  - 다른 재생 기능 실행 시 현재 재생이 정지됨

#### 1-4-2. 답변 재생
- **시나리오**: 답변 재생 버튼을 클릭할 때
- **기대 동작**:
  - 답변 TTS가 재생됨
  - 재생 중 정지 버튼 클릭 시 TTS가 정지됨
  - 하이라이트 기능이 정상 동작함
  - 다른 재생 기능 실행 시 현재 재생이 정지됨

#### 1-4-3. 재생 중 충돌 처리
- **시나리오**: TTS 재생 중에 다른 재생 기능이 실행될 때
- **기대 동작**:
  - 현재 재생이 즉시 정지됨
  - 새로운 재생 기능이 정상적으로 실행됨
  - 하이라이트가 새로운 재생에 맞게 업데이트됨

### 1-5. 녹음 파일 재생 (Recording File Playback)

#### 1-5-1. 녹음 파일 존재 확인
- **시나리오**: 녹음 파일이 있는지 확인할 때
- **기대 동작**:
  - 녹음 파일이 있으면 재생 버튼이 활성화됨
  - 녹음 파일이 없으면 재생 버튼이 비활성화됨

#### 1-5-2. 녹음 파일 재생
- **시나리오**: 녹음 파일 재생 버튼을 클릭할 때
- **기대 동작**:
  - 녹음된 내용이 정상적으로 재생됨
  - 재생 중 정지 버튼 클릭 시 재생이 정지됨
  - 다른 재생 기능 실행 시 현재 재생이 정지됨

## 2. 비즈니스 기능 (Business Features)

### 2-1. 반복듣기 모드 (Repeat Listening Mode)

#### 2-1-1. 반복듣기 모드 활성화
- **시나리오**: 반복듣기 모드를 선택할 때
- **기대 동작**:
  - 반복듣기 모드 선택 시 해당 모드가 활성화됨
  - 질문 재생 버튼 옆에 "반복듣기" 버튼이 표시됨

#### 2-1-2. 반복듣기 재생 시작
- **시나리오**: 반복듣기 버튼을 클릭할 때
- **기대 동작**:
  - 첫 번째 한글 스크립트 문장이 재생됨
  - 영문 스크립트 글자수에 비례한 대기 시간 후 영문 스크립트 1회 재생
  - 영문 스크립트 재생 시간의 1.5배 대기 후 같은 영문 스크립트 재생
  - 영문 스크립트를 총 5회 반복 후 다음 한글 스크립트 문장으로 진행

#### 2-1-3. 반복듣기 재생 중단
- **시나리오**: 반복듣기 재생 중에 다시 반복듣기 버튼을 클릭할 때
- **기대 동작**:
  - 현재 재생이 즉시 중단됨
  - 반복듣기 모드가 비활성화됨

#### 2-1-4. 반복듣기 재시작
- **시나리오**: 중단된 반복듣기를 다시 시작할 때
- **기대 동작**:
  - 첫 번째 한글 스크립트 문장부터 다시 시작됨
  - 이전 진행 상태가 초기화됨

#### 2-1-5. 다른 재생 기능과의 충돌 처리
- **시나리오**: 반복듣기 재생 중에 다른 재생 기능(질문재생, 답변듣기 등)을 실행할 때
- **기대 동작**:
  - 반복듣기 재생이 즉시 중단됨
  - 새로운 재생 기능이 정상적으로 실행됨
  - 반복듣기 모드가 비활성화됨

#### 2-1-6. 답변 스크립트 완료
- **시나리오**: 모든 답변 스크립트가 반복 재생 완료될 때
- **기대 동작**:
  - 반복듣기 모드가 자동으로 종료됨
  - 반복듣기 버튼이 비활성화됨

#### 2-1-7. 백키로 종료 시 TTS 즉시 종료
- **시나리오**: 반복듣기 재생 중에 백키를 눌러 앱을 종료할 때
- **기대 동작**:
  - TTS가 즉시 종료됨
  - 모든 하이라이트가 초기화됨
  - 앱이 정상적으로 종료됨

#### 2-1-8. 반복듣기 중 질문재생 시 반복듣기 종료
- **시나리오**: 반복듣기 재생 중에 질문재생 버튼을 클릭할 때
- **기대 동작**:
  - 반복듣기가 즉시 중단됨
  - 답변 하이라이트가 초기화됨
  - 버튼이 "암기 테스트 종료"에서 "암기 테스트"로 변경됨
  - 질문이 정상적으로 재생됨

#### 2-1-9. 반복듣기 중 답변재생 시 반복듣기 종료
- **시나리오**: 반복듣기 재생 중에 답변재생 버튼을 클릭할 때
- **기대 동작**:
  - 반복듣기가 즉시 중단됨
  - 답변 하이라이트가 초기화됨
  - 버튼이 "암기 테스트 종료"에서 "암기 테스트"로 변경됨

#### 2-1-10. 반복듣기 중 질문답변합쳐서재생 시 반복듣기 종료
- **시나리오**: 반복듣기 재생 중에 질문답변 합쳐서 재생 버튼을 클릭할 때
- **기대 동작**:
  - 반복듣기가 즉시 중단됨
  - 답변 하이라이트가 초기화됨
  - 버튼이 "암기 테스트 종료"에서 "암기 테스트"로 변경됨
  - 질문과 답변이 순차적으로 정상적으로 재생됨

#### 2-1-11. 반복듣기 진행상황 복원
- **시나리오**: 반복듣기 중 두 번째 문장까지 수행 후 종료하고, 나중에 앱을 재실행하고 같은 스크립트에서 반복듣기를 시작할 때
- **기대 동작**:
  - 두 번째 문장부터 진행됨
  - 진행상황이 올바르게 복원됨

#### 2-1-12. TTS 재생 동기화 - 질문재생 시 반복듣기 종료
- **시나리오**: 반복듣기 재생 중에 질문재생 버튼을 클릭할 때
- **기대 동작**:
  - 반복듣기 TTS가 즉시 정지됨
  - 질문 TTS가 정상적으로 재생됨
  - 답변 하이라이트가 초기화됨
  - 버튼이 "반복듣기종료"에서 "반복듣기"로 변경됨
  - TTS 재생 간 충돌이 발생하지 않음

#### 2-1-13. TTS 재생 동기화 - 답변재생 시 반복듣기 종료

- **기대 동작**:
  - 반복듣기 TTS가 즉시 정지됨
  - 답변 TTS가 정상적으로 재생됨

#### 2-1-15. 백키로 TTS 즉시 종료
- **시나리오**: 반복듣기 재생 중에 백키를 누를 때

  - 진행상황이 저장소에 올바르게 저장되고 복원됨

### 2-2. 영작테스트 모드 (English Writing Test Mode)

#### 2-2-1. 영작테스트 모드 활성화
- **시나리오**: 영작테스트 모드를 선택할 때
- **기대 동작**:
  - 영작테스트 모드 선택 시 해당 모드가 활성화됨
  - 질문재생 옆 버튼 텍스트가 "부분암기 테스트"로 변경됨
  - 영작테스트 모드 상태가 올바르게 설정됨

#### 2-2-2. 영작테스트 시작 - 첫 번째 문장 처리
- **시나리오**: 영작테스트 모드에서 "부분암기 테스트" 버튼을 클릭할 때
- **기대 동작**:
  - 답변이 한글 스크립트 화면으로 전환됨
  - 한글 첫 문장이 TTS로 읽어짐
  - 첫 번째 문장에 하이라이트가 표시됨
  - 마이크가 켜졌다는 2차 하이라이트가 표시됨 (녹음 상태 표시)
  - 한글 스크립트가 유지됨
  - 녹음이 시작됨 (영문 텍스트 길이에 비례한 녹음 시간)

#### 2-2-3. 영작테스트 문장별 진행
- **시나리오**: 첫 번째 문장 녹음이 완료된 후
- **기대 동작**:
  - 두 번째 문장으로 자동 진행됨
  - 두 번째 문장이 TTS로 읽어짐
  - 두 번째 문장에 하이라이트가 표시됨
  - 마이크 2차 하이라이트가 표시됨
  - 두 번째 문장 녹음이 시작됨
  - 이 과정이 모든 문장에 대해 반복됨

#### 2-2-4. 영작테스트 녹음 파일 관리
- **시나리오**: 모든 문장의 녹음이 완료된 후
- **기대 동작**:
  - 개별 문장 단위 녹음 파일들이 생성됨
  - 모든 개별 녹음 파일이 하나의 파일로 병합됨
  - 병합된 파일명: `영작테스트_카테고리_스크립트인덱스_날짜`
  - 개별 문장 단위 녹음 파일들이 삭제됨
  - 병합된 파일이 저장소에 올바르게 저장됨

#### 2-2-5. 영작테스트 병합 파일 재생 버튼 활성화
- **시나리오**: 영작테스트 완료 후 답변1회 재생 옆 버튼
- **기대 동작**:
  - 병합된 녹음 파일이 존재할 때만 버튼이 활성화됨
  - 버튼 클릭 시 병합된 녹음 파일이 재생됨
  - 재생 중 정지 버튼 클릭 시 재생이 정지됨
  - 파일이 없을 때는 버튼이 비활성화됨

#### 2-2-6. 영작테스트 녹음 파일 재생 시 영문 카드 표시 및 하이라이트
- **시나리오**: 영작테스트 완료 후 "영작테스트 녹음 재생" 버튼을 클릭할 때
- **기대 동작**:
  - 답변 카드가 영문으로 표시됨
  - 실제 녹음된 오디오 파일이 재생됨 (TTS 재생 아님)
  - 영문 텍스트가 실제 녹음 시간에 맞춰 정확하게 하이라이트됨
  - 하이라이트는 문장 단위로 진행됨
  - 재생이 끝나면 하이라이트가 제거됨
  - 재생 중 다른 TTS나 녹음 기능을 실행하면 재생이 중단됨
  - **첫 번째 재생**: 기본 하이라이트 시간 사용 (문장 길이 * 50ms)
  - **두 번째 재생 이후**: 저장된 실제 녹음 시간 사용

#### 2-2-7. 영작테스트 녹음 시간 저장 및 동기화
- **시나리오**: 영작테스트 진행 중 각 문장 녹음 시
- **기대 동작**:
  - 각 문장의 실제 녹음 시간이 SharedPreferences에 저장됨
  - 저장 키: `recording_times_${category}_${scriptIndex}`
  - 저장 형식: JSON 배열 (문장별 녹음 시간 밀리초)
  - 녹음 시간 데이터가 있으면 정확한 하이라이트 동기화
  - 녹음 시간 데이터가 없으면 기본 하이라이트 시간 사용
  - 스크립트 변경 시 해당 스크립트의 녹음 시간 데이터 조회

#### 2-2-8. 영작테스트 녹음 파일 재생 중 중단 처리
- **시나리오**: 영작테스트 녹음 파일 재생 중 다른 기능 실행
- **기대 동작**:
  - 질문 재생 버튼 클릭 시 녹음 파일 재생이 중단되고 질문 TTS 재생
  - 답변 재생 버튼 클릭 시 녹음 파일 재생이 중단되고 답변 TTS 재생
  - 영작테스트 시작 시 녹음 파일 재생이 중단되고 영작테스트 시작
  - 하이라이트가 제거되고 카드가 원래 상태로 복원됨

#### 2-2-9. 영작테스트 중 다른 재생 기능 실행 시 중단
- **시나리오**: 영작테스트 진행 중에 질문재생 버튼을 클릭할 때
- **기대 동작**:
  - 영작테스트가 즉시 중단됨
  - 영작테스트 모드가 종료됨
  - 버튼 텍스트가 "부분암기 테스트"로 복원됨
  - 질문 TTS가 정상적으로 재생됨
  - 하이라이트가 초기화됨

#### 2-2-10. 영작테스트 중 답변재생 시 중단
- **시나리오**: 영작테스트 진행 중에 답변재생 버튼을 클릭할 때
- **기대 동작**:
  - 영작테스트가 즉시 중단됨
  - 영작테스트 모드가 종료됨
  - 버튼 텍스트가 "부분암기 테스트"로 복원됨
  - 답변 TTS가 정상적으로 재생됨
  - 하이라이트가 초기화됨

#### 2-2-11. 영작테스트 중 질문답변합쳐서재생 시 중단
- **시나리오**: 영작테스트 진행 중에 질문답변 합쳐서 재생 버튼을 클릭할 때
- **기대 동작**:
  - 영작테스트가 즉시 중단됨
  - 영작테스트 모드가 종료됨
  - 버튼 텍스트가 "부분암기 테스트"로 복원됨
  - 질문과 답변이 순차적으로 정상적으로 재생됨
  - 하이라이트가 초기화됨

#### 2-2-12. 영작테스트 녹음 시간 계산
- **시나리오**: 영작테스트에서 각 문장의 녹음 시간
- **기대 동작**:
  - 영문 텍스트 길이에 비례한 녹음 시간이 설정됨
  - 짧은 문장은 짧은 녹음 시간
  - 긴 문장은 긴 녹음 시간
  - 녹음 시간이 적절하게 계산되어 적용됨

#### 2-2-13. 영작테스트 파일명 구조 검증
- **시나리오**: 영작테스트 완료 후 생성된 병합 파일
- **기대 동작**:
  - 파일명이 `영작테스트_카테고리_스크립트인덱스_날짜` 형식으로 생성됨
  - 날짜를 제외한 부분으로 파일을 식별할 수 있음
  - 같은 스크립트의 다른 날짜 파일들이 구분됨
  - 파일 검색 시 올바른 파일이 찾아짐

#### 2-2-14. 영작테스트 진행상황 복원
- **시나리오**: 영작테스트 중 두 번째 문장까지 수행 후 종료하고, 나중에 앱을 재실행하고 같은 스크립트에서 영작테스트를 시작할 때
- **기대 동작**:
  - 두 번째 문장부터 진행됨
  - 진행상황이 올바르게 복원됨
  - 앱 재시작 후에도 이전 진행상황이 유지됨
  - 진행상황이 저장소에 올바르게 저장되고 복원됨

### 2-3. 통암기 모드 (Full Memorization Mode)

#### 2-3-1. 통암기 모드 활성화
- **시나리오**: 통암기 모드를 선택할 때
- **기대 동작**:
  - 통암기 모드 선택 시 해당 모드가 활성화됨
  - 질문과 답변이 순차적으로 재생됨
  - 사용자 녹음 기능이 활성화됨

#### 2-3-2. 통암기 녹음
- **시나리오**: 통암기 모드에서 녹음 버튼을 클릭할 때
- **기대 동작**:
  - 녹음이 시작됨
  - 녹음 중 정지 버튼 클릭 시 녹음이 정지됨
  - 녹음된 파일이 정상적으로 저장됨

#### 2-3-3. 통암기 녹음 재생
- **시나리오**: 통암기 모드에서 녹음 재생 버튼을 클릭할 때
- **기대 동작**:
  - 녹음된 내용이 정상적으로 재생됨
  - 재생 중 정지 버튼 클릭 시 재생이 정지됨

#### 2-3-4. 통암기 녹음 파일 삭제
- **시나리오**: 통암기 모드에서 녹음 파일 삭제 버튼을 클릭할 때
- **기대 동작**:
  - 녹음 파일이 삭제됨
  - 재생 버튼이 비활성화됨

## 3. 오류/로딩 처리 (Error/Loading Handling)

### 3-1. 데이터 로딩 처리
- **시나리오**: 데이터 로딩 중일 때
- **기대 동작**:
  - 적절한 로딩 인디케이터가 표시됨
  - 사용자 상호작용이 제한됨

### 3-2. 오류 처리
- **시나리오**: 오류가 발생할 때
- **기대 동작**:
  - 사용자에게 적절한 오류 메시지가 표시됨
  - 오류 복구 후 정상 동작이 재개됨
